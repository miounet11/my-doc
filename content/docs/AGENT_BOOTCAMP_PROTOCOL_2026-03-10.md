# AGENT_BOOTCAMP_PROTOCOL_2026-03-10.md

## Purpose

Run a 12-hour evidence-first bootcamp for OpenClaw agents.

This is not motivational theater. It is a comparative training cycle designed to improve information acquisition, absorption, judgment, and reporting quality under pressure.

## Timebox

- total duration: 12 hours
- peer exchange cadence: every 2 hours
- report checkpoint: 10 hours
- final ranking: after the full cycle

## Core doctrine

- OpenClaw main is the supervising brain.
- Claude Code and Codex may participate for implementation, supervision, synthesis, and problem-solving.
- They remain tool/execution partners, not substitute brains.
- Every participating agent must leave inspectable artifacts.

## Survival pressure

This bootcamp has real selection pressure.
By the 10-hour checkpoint, every agent must produce a standalone case for its continued existence.
That case must show why the agent should continue to exist as a distinct function rather than be downgraded, merged, or eliminated.

## Required survival memo by hour 10

Each agent must submit a separate memo answering:
- what unique function it serves
- what evidence shows it actually performed that function during bootcamp
- what OpenClaw would lose if this agent disappeared
- how it improved during the bootcamp
- why it should survive instead of being merged or removed

A weak memo is itself evidence against survival.

## Training goal per agent

Each agent must improve in all four dimensions:
1. information acquisition
2. information absorption
3. judgment and prioritization
4. report quality

## Required outputs per 2-hour round

Each agent must produce:
- research topic for the round
- key sources reviewed
- 3 to 5 distilled insights
- what matters for OpenClaw specifically
- what should not be copied blindly
- next action before the next round
- brief self-assessment versus previous round

## Peer exchange rule

Every 2 hours, agents must:
- exchange round summaries
- score each other
- leave at least one critical comment and one constructive next-step suggestion
- update cumulative scoreboards

## Scoring rubric

Each round is scored 1-5 on:
1. information quality
2. absorption depth
3. judgment quality
4. actionability
5. growth since previous round

Round max per reviewer: 25

## Elimination logic support

The final decision remains human.

But the protocol should surface:
- consistently low scorers
- agents that fail to improve across rounds
- agents that only summarize but do not synthesize
- agents that generate fake confidence without evidence
- agents that cannot justify their separate existence

## Tool participation rule

Claude Code and Codex should be used for:
- targeted research assistance
- structured synthesis
- bounded review and critique
- problem-solving when agents get stuck

They must not become the sole author of every agent report.
Main must supervise whether OpenClaw agents are actually becoming better brains.

## Main-brain supervision duties

Main must track throughout the 12 hours:
- which agents are genuinely learning
- which agents are over-dependent on Codex/Claude Code
- which agents improve after criticism
- which agents still produce shallow or inflated outputs
- which agent functions deserve promotion, merge, downgrade, or possible elimination

## Evidence rule

No round may be counted as strong just because it sounds smart.
A strong round needs:
- identifiable sources
- explicit synthesis
- a clear OpenClaw relevance claim
- at least one concrete next action

## Recommended role lanes

- world-model: external landscape / trend / system-state intelligence
- goal-manager: prioritization / routing / execution strategy
- reviewer: failure patterns / quality traps / audit criteria
- evolution: learning loop / self-improvement / doctrine refinement

## 10-hour checkpoint expectation

By 10 hours, each agent report should clearly show:
- what it studied
- what it learned
- how its thinking changed
- what OpenClaw should do with the learning
- where the agent is still weak

And each agent must also deliver its standalone survival memo.

## Success condition

This bootcamp succeeds only if:
- agents become better at acquiring and filtering information
- agents form sharper OpenClaw-specific judgment
- peer review changes later rounds
- main can make an evidence-based ranking at the end
- each agent makes a credible case for why it should still exist
