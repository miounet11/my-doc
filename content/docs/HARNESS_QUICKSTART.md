# Harness Quickstart

Use this as the shortest entrypoint when you need the harness fast.

## If the task is...
- heartbeat/self-check -> read `skills/heartbeat-closure/SKILL.md` and use `heartbeat` profile from `docs/RUNTIME_PROFILES.md`
- finishing/reporting a meaningful task -> read `skills/task-closure/SKILL.md`
- claiming mydoc publish/delivery -> read `skills/mydoc-publish-verification/SKILL.md`
- reconstructing earliest context/history -> read `skills/context-recovery/SKILL.md` and usually start in `explore`
- implementing meaningful changes -> use `implement`
- validating whether work actually satisfies the ask -> read `skills/verification-agent/SKILL.md` and use `verify`
- diagnosing a blocked/degraded state -> use `recover`

## Fast Rule
If uncertain, choose the lower honest completion/status level and say what remains unverified.
