# 网站 GEO / SEO 发布清单（2026 可执行版）

## 文档目的

这是一份给运营、内容、开发直接照着做的 **网站 GEO / SEO 发布清单**。

目标不是讲概念，而是确保每一篇要发布的页面、每一个重要落地页，都同时满足：
- 搜索引擎可抓取
- AI 搜索可理解/可引用
- 内容结构清晰
- 信任信号足够
- 后续可持续扩展

适用页面：
- 首页
- 产品页
- 功能页
- 博客文章
- 教程页
- FAQ 页
- 对比页
- 文档页

---

# 一、发布前先做 5 个底层检查（技术底座）

## 1. 页面能否被正常访问
- [ ] 页面返回 200，不是 3xx/4xx/5xx 异常状态
- [ ] 页面不是登录后才能看
- [ ] 页面不是空白壳子，需要 JS 很久后才出现正文
- [ ] 首屏 HTML 里能直接看到主要内容

## 2. 页面是否允许被抓
- [ ] `robots.txt` 没有误封该页面
- [ ] 关键 CSS / JS / 图片资源没有被 robots 阻断
- [ ] 页面没有错误的 `noindex`
- [ ] 站点有 `sitemap.xml`
- [ ] 页面 URL 已加入 sitemap（重要页建议加入）

## 3. URL 是否规范
- [ ] URL 简短、可读、包含主题词
- [ ] 没有多个重复 URL 对应同一内容
- [ ] canonical 指向正确
- [ ] 中英文 slug 规则统一

## 4. 页面能否被站内发现
- [ ] 至少有一个站内入口链接到它
- [ ] 它属于明确的栏目/主题
- [ ] 它不是孤岛页

## 5. AI crawler 是否没被误伤
重点检查这些 user-agent 的策略：
- [ ] OAI-SearchBot
- [ ] GPTBot
- [ ] ChatGPT-User（了解用途）
- [ ] Googlebot
- [ ] Google-Extended
- [ ] 其他你关心的 crawler

说明：
- 想进 ChatGPT 搜索结果，别误封 `OAI-SearchBot`
- 不想训练用途，不等于不想搜索可见，要区分 `GPTBot` 与搜索 crawler

---

# 二、每个页面发布前必须满足的内容结构

## 6. 页面是否“一开头就回答问题”
- [ ] 开头 1~3 段内直接说明这页讲什么
- [ ] 用户不用滚很久才看到答案
- [ ] 没有大段空话、套话、营销废话挡在前面

## 7. 标题是否清晰
- [ ] Title 明确，不堆关键词
- [ ] H1 和页面核心主题一致
- [ ] 标题不是标题党，不夸张

## 8. 是否具备 AI / 搜索友好的答案块
至少具备其中 2~4 种：
- [ ] 定义块
- [ ] 步骤块
- [ ] FAQ 块
- [ ] 对比块
- [ ] checklist 块
- [ ] 常见错误块
- [ ] 适用 / 不适用块

## 9. heading 结构是否清楚
- [ ] H1 只有一个
- [ ] H2 / H3 逻辑清晰
- [ ] 不跳级、不乱嵌套
- [ ] 小标题是“有信息量”的，不是空泛词

## 10. 页面是否真的有独特价值
- [ ] 不是简单改写别人的内容
- [ ] 有原创经验、原创判断、原创整理或原创案例
- [ ] 读完后用户不用立刻再去搜一遍同样问题

---

# 三、信任与实体信息检查（很重要）

## 11. 作者/来源清楚吗
- [ ] 有作者名或团队名
- [ ] 有更新时间
- [ ] 专业内容最好有作者简介或审核人信息
- [ ] 有明确来源/引用时，标出出处

## 12. 网站整体可信度支撑是否存在
- [ ] About 页面可访问
- [ ] 联系方式存在
- [ ] 公司/团队身份明确
- [ ] 站点主题聚焦，不是杂乱内容农场

## 13. 对 YMYL 类内容加倍谨慎
如果涉及：
- 健康
- 医疗
- 金融
- 法律
- 安全

则必须额外检查：
- [ ] 是否有专业来源
- [ ] 是否避免武断结论
- [ ] 是否有风险提示
- [ ] 是否有明显事实错误

---

# 四、结构化数据 / schema 检查

## 14. 至少检查这些 schema 类型
按页面类型选：
- [ ] Organization
- [ ] WebSite
- [ ] BreadcrumbList
- [ ] Article / BlogPosting
- [ ] FAQPage
- [ ] Product
- [ ] LocalBusiness
- [ ] SoftwareApplication

## 15. schema 的质量要求
- [ ] 与页面可见内容一致
- [ ] 不填假数据
- [ ] 必填字段完整
- [ ] 不为了 rich result 乱塞不相关 schema
- [ ] 以 JSON-LD 为主

---

# 五、发布什么内容最容易被 AI 和搜索引擎喜欢

优先级最高的内容类型：

## 16. 定义型内容
例如：
- 什么是 GEO
- 什么是 robots.txt
- 什么是 canonical

检查：
- [ ] 一句话定义明确
- [ ] 有详细展开
- [ ] 有适用场景

## 17. 教程型内容
例如：
- 如何配置 AI crawler 访问策略
- 如何给产品页加 schema

检查：
- [ ] 有前提条件
- [ ] 有步骤
- [ ] 有验证方法
- [ ] 有常见错误

## 18. 对比型内容
例如：
- GEO vs SEO
- ChatGPT Search vs Google AI Overviews

检查：
- [ ] 核心差异写清楚
- [ ] 适用场景写清楚
- [ ] 最后给建议而不是只列点

## 19. FAQ / 问题型内容
例如：
- AI 搜索会看 robots.txt 吗
- FAQ schema 还有用吗

检查：
- [ ] 问题真实存在
- [ ] 答案简洁直接
- [ ] 不是为了凑长尾硬写奇怪问题

## 20. 案例 / 改造前后内容
例如：
- 产品页如何改成 AI 可引用结构
- 站点 schema 补全前后差异

检查：
- [ ] 背景清楚
- [ ] 改了什么清楚
- [ ] 结果不要夸大
- [ ] 最好有截图/示例

## 21. checklist / best practices 内容
例如：
- GEO 上线清单
- SaaS 官网 SEO/GEO 检查清单

检查：
- [ ] 能让人直接执行
- [ ] 条目清楚，不空泛
- [ ] 有优先级

---

# 六、自动扩展内容的正确方法

## 22. 不要按“关键词堆量”扩展
避免：
- [ ] 同义词换壳批量发
- [ ] 追热点乱写
- [ ] 一天几十篇没主题关系的内容

## 23. 应按“问题树”扩展
围绕一个核心主题，拆成：
- [ ] 定义
- [ ] 教程
- [ ] FAQ
- [ ] 对比
- [ ] 错误/避坑
- [ ] 案例
- [ ] 清单

## 24. 每篇内容发布后都要能衍生下一批题目
发布后问：
- [ ] 这篇里的哪个 H2 值得单独展开？
- [ ] 用户下一步会问什么？
- [ ] 哪个 FAQ 可以独立成文章？
- [ ] 哪个案例值得补？

## 25. 给内容类型建立固定模板
建议为以下内容建立模板：
- [ ] 教程模板
- [ ] FAQ 模板
- [ ] 对比模板
- [ ] 案例模板
- [ ] checklist 模板

---

# 七、内部链接与主题集群

## 26. 每篇页面都要在主题树里有位置
- [ ] 它属于哪个主题集群？
- [ ] 它支撑哪一个核心页？
- [ ] 它应该链接哪些相关页？

## 27. 发布前检查内链
- [ ] 至少链向 2~5 个相关页面
- [ ] 有上级页 / 下级页关系
- [ ] 没有一堆“点击这里”这种无信息锚文本

---

# 八、发布后运营检查

## 28. 发布后 24~72 小时检查
- [ ] 页面能打开
- [ ] schema 没坏
- [ ] canonical 正常
- [ ] robots 没误伤
- [ ] 标题/描述没被模板覆盖错

## 29. 周度检查
- [ ] 哪些页有流量增长
- [ ] 哪些页没有被收录/抓取
- [ ] 哪些页缺 FAQ / 对比 / 作者信息
- [ ] 哪些旧页需要更新

## 30. 月度检查
- [ ] 哪个主题集群最强
- [ ] 哪个主题集群内容太薄
- [ ] 哪些内容只是有流量没转化
- [ ] 哪些内容值得升级成核心页/案例页/工具页

---

# 九、绝对不要做的事

- [ ] 不要把 GEO 当成“AI 批量洗稿”
- [ ] 不要为了新鲜感乱改发布日期
- [ ] 不要为了 rich result 硬塞假 schema
- [ ] 不要只做博客，不做核心落地页
- [ ] 不要为了搜索流量进入自己完全不懂的领域
- [ ] 不要把 robots.txt 当成真正的防索引手段
- [ ] 不要只追求内容数量，不验证页面质量

---

# 十、推荐发布顺序（最实用）

如果资源有限，按这个顺序做：

## P0
- 首页
- 产品页
- 功能页
- FAQ
- sitemap / robots / canonical / schema 基础层

## P1
- 核心教程页
- 对比页
- 常见错误页
- checklist 页

## P2
- 案例页
- 文档页
- 行业专题页
- 自动内容扩展系统

---

# 十一、最终标准：一篇页面值不值得发

发布前最后问 7 个问题：

1. 这页有没有直接回答真实问题？
2. 这页是不是比现有结果多一点真实价值？
3. 这页是不是结构清楚、可抓取、可理解？
4. 这页有没有作者/品牌/可信度支撑？
5. 这页有没有可能被 AI 摘取成答案块？
6. 这页是不是在我们的主题体系里有位置？
7. 这页发出去后，用户读完还会不会立刻去搜同一个问题？

如果前 6 个都还行，第 7 个答案是“不会”，这篇通常就值得发。
